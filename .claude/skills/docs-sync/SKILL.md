# /docs-sync — Сверка документации с кодом

Выполни все 5 проверок последовательно и выведи финальный отчёт. Работай в директории проекта.

## 1. Упоминаемые файлы существуют

Собери все пути файлов, упомянутые в `docs/*.md` и `CLAUDE.md`. Ищи паттерны вида:
- `src/...` (компоненты, сторы, сервисы и т.д.)
- `data/...` (JSON-файлы данных)
- `eslint.config.*`, `prettier.config.*`, `.prettierrc*`, `tsconfig.*`
- `firebase.json`, `vite.config.*`
- Любые другие пути к файлам/директориям проекта

Используй Grep для поиска путей в документации:
```
pattern: "(src/[a-zA-Z0-9_./-]+|data/[a-zA-Z0-9_./-]+|[a-z]+\.config\.[a-z]+|\.prettierrc[a-z.]*|firebase\.json|package\.json)"
glob: "*.md"
path: "docs/"
```

Также проверь пути в `CLAUDE.md`.

Для каждого найденного пути проверь его существование через Glob. Игнорируй:
- Пути с подстановочными знаками (`*.vue`, `**/*.ts`)
- URL-адреса (`http://`, `https://`)
- Пути из примеров кода и шаблонов

Результат:
- ✅ Все упомянутые файлы найдены
- ❌ Файл `X` упомянут в `Y`, но не существует (для каждого отсутствующего)

## 2. Количество строк в компонентах

Найди в `docs/*.md` упоминания количества строк (паттерны вроде `N строк`, `N lines`, `~N строк`). Используй Grep:
```
pattern: "\d+\s*(строк|lines|стр\.)"
glob: "*.md"
path: "docs/"
```

Для каждого найденного упоминания определи файл, к которому оно относится, и сверь с фактическим количеством строк:
```bash
wc -l <путь_к_файлу>
```

Также независимо проверь ключевые файлы:
```bash
wc -l src/pages/*.vue src/components/*.vue src/stores/resource/index.ts src/services/http.ts
```

Результат:
- ✅ Количество строк совпадает (±10%)
- ⚠️ Расхождение >10%: документ `X` указывает N строк, фактически M строк в файле `Z`

## 3. Технологии и версии в package.json

Прочитай `package.json` с помощью Read.

Найди таблицы технологий в `CLAUDE.md` и `docs/01-project-overview.md` (ищи строки таблиц с `|`).

Для каждой технологии из таблиц проверь:
- Пакет присутствует в `dependencies` или `devDependencies` в `package.json`
- Указанная в документации версия примерно соответствует фактической (совпадает мажорная версия)

Также проверь обратное: нет ли в документации пакетов, которые уже удалены из `package.json` (например `firebase`, `xlsx`).

Результат:
- ✅ Все технологии и версии соответствуют
- ❌ Пакет `X` упомянут в документации, но отсутствует в `package.json`
- ❌ Пакет `X` в документации указан как `vA`, фактически `vB` (если отличается мажорная версия)

## 4. Статусы задач рефакторинга

Прочитай `docs/15-refactoring-progress.md` и `docs/02-refactoring-priorities.md` с помощью Read.

Проверь согласованность:
- Статусы фаз (Завершена / В процессе / Не начата) в doc 02 соответствуют статусам в doc 15
- Если фаза помечена как «Завершена» в одном документе, она должна быть «Завершена» и в другом
- Чекбоксы `[x]` / `[ ]` в doc 15 не противоречат общему статусу фазы

Результат:
- ✅ Статусы согласованы между документами
- ❌ Фаза `X` имеет статус `A` в doc 02, но `B` в doc 15
- ❌ Фаза `X` помечена как завершённая, но содержит незавершённые чекбоксы `[ ]`

## 5. Структура проекта

Найди деревья файлов/директорий в `docs/01-project-overview.md` и `CLAUDE.md` (блоки с отступами и `├──`, `└──`, `│` или просто списки директорий `src/...`).

Проверь через Glob:
- Все директории из дерева существуют
- Все файлы из дерева существуют

Также проверь обратное — нет ли существенных файлов/директорий в `src/`, не отражённых в документации:
```
Glob: src/*/ — директории первого уровня
Glob: src/**/*.vue — все Vue-компоненты
Glob: src/**/*.ts — все TypeScript-файлы (кроме .d.ts)
```

Игнорируй при обратной проверке:
- `*.d.ts` файлы
- `env.d.ts`, `shims-*.d.ts`
- Файлы в `node_modules/`

Результат:
- ✅ Структура в документации соответствует фактической
- ❌ Директория/файл `X` указана в документации, но не существует
- ⚠️ Файл/директория `X` существует в `src/`, но не отражена в документации
