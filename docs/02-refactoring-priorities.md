## Принципы работы

1. **Каждый этап — отдельная ветка и PR** в `main`
2. **Не смешивать фазы** в одном PR
3. **Сохранение функционала и внешнего вида**: при рефакторинге функциональность и визуальное оформление приложения должны быть **полностью сохранены**. Ни один пользовательский сценарий не должен сломаться, ни один элемент UI не должен измениться визуально
4. **Лимит 300 строк**: ни один файл в проекте не должен превышать 300 строк. Планы декомпозиции всех крупных файлов описаны в [doc 06](06-resource-plan-decomposition.md) (ResourcePlan) и [doc 14](14-pages-decomposition.md) (остальные страницы и компоненты)
5. **Регрессионная проверка**: после каждого PR убедиться, что приложение работает (ручное тестирование — тесты не настроены). Использовать чеклист ниже
6. **Порядок внутри фазы** не строгий — задачи одной фазы можно делать параллельно
7. **Фиксация результатов**: по завершении каждой фазы заполнять отчёт в [15-refactoring-progress.md](15-refactoring-progress.md)

- [ ] Переключение режимов отображения (total / quarterly / split) работает
- [ ] Редактирование ячеек — ввод часов сохраняется
- [ ] Квартальная разбивка (q1–q4) корректно пересчитывается
- [ ] Итоги по строкам и столбцам считаются правильно
- [ ] Диаграммы загрузки мощностей отображаются
- [ ] Фильтрация по заказчику и менеджеру работает
- [ ] Показ/скрытие групп работает
- [ ] Экспорт в CSV работает
- [ ] Горизонтальный скролл работает

### Страница «Проекты» (`/projects`)
