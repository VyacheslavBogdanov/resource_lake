# Доработки ресурсного плана

- [ ] **1.** Ранее мы в Excel фиксировали ошибки планирования. На сайте нет возможности указать, сколько реально потрачено, для анализа.
    - **Решение:** **Где:** «Управление данными» → таблица ввода (поле рядом с плановыми часами); «Ресурсный план» → новые колонки в таблице. На «Управление данными» рядом с плановыми часами появится поле для ввода фактически потраченных часов, а на «Ресурсном плане» добавятся колонка «Факт» и колонка отклонения (план минус факт) — будет видно, где ресурсы недо- или переоценили. **Оценка: 16 ч**
- [ ] **2.** Дата обновления данных по ресурсу
    - **Решение:** **Где:** «Ресурсный план» → ячейки распределения (подсказка при наведении на ячейку или отдельная колонка); то же рядом с полем на «Управление данными». Возле каждого значения ресурса будет видно дату его последнего изменения — сразу понятно, насколько свежие данные. **Оценка: 6 ч**
- [ ] **3.** Добавить систему стимуляции руководителя проекта поддерживать план в актуальном состоянии
    - **Решение:** **Где:** «Ресурсный план» → колонка «Проект» (подсветка) и сводка над таблицей, рядом с показателями загрузки. Проекты с устаревшими данными будут подсвечиваться цветом, а сверху появится сводка «кто обновил план, а кто нет». Это подталкивает руководителей держать план актуальным. Пересекается с пунктами 9 и 13. **Оценка: 8 ч**
- [ ] **4.** Фильтрация строк с нулевыми значениями для выбранной группы/ресурса. Проектов много, но значимых строк бывает менее десятка — не в кайф скроллить пустые строки
    - **Решение:** _Нужно пояснение Заказчика_
- [ ] **5.** На странице «Ресурсный план», столбец «доступно» не информативен. Лучше указывать часы, которые на данный момент доступны
    - **Решение:** **Где:** «Ресурсный план» → шапка таблицы, подпись «доступно: … ч» под каждой колонкой группы. Вместо общей мощности группы будет показываться, сколько часов ещё свободно (мощность минус уже распределённые). Перегруз (распределили больше, чем есть) подсветится. **Оценка: 4 ч**
- [ ] **6.** На вкладке «Ресурсный план» в столбце «Проект» добавить возможность выбора определенных проектов (выбор осуществляет пользователь) для отображения только их
    - **Решение:** **Где:** Вариант 1: «Ресурсный план» → панель фильтров над таблицей (рядом с фильтрами «Заказчик» / «Руководитель проекта»). Появится возможность галочками выбрать конкретные проекты и видеть в таблице только их. Вариант 2: Сделать отдельный фильтр в ячейке "Проект" **Оценка: 6 ч**
- [ ] **7.** На вкладке «Ресурсный план», при наведении на проект выскакивает хинт, в нем помимо заказчика еще указывать и руководителя проекта.
    - **Решение:** **Где:** «Ресурсный план» → колонка «Проект», всплывающая подсказка при наведении на название проекта. В подсказке, помимо заказчика, будет показываться и руководитель проекта. Сейчас дефолтные тултипы, взять кастомные из UI Kit? **Оценка: 1 ч**
- [ ] **8.** Автоматическое сохранение введенных данных на странице «Управление данными», без нажатия кнопки «Сохранить»
    - **Решение:** **Где:** «Управление данными» → таблица ввода и область рядом с кнопкой «Сохранить» в панели сверху. Введённые данные будут сохраняться сами, без нажатия кнопки; рядом появится индикатор «Сохранение… / Сохранено», Кнопку "Сохранить изменения" удалить. **Оценка: 8 ч**
- [ ] **9.** Предлагается ввести помесячную фиксацию актуальности ресурсного плана внутри каждого квартала: руководитель проекта вручную подтверждает, что данные за конкретный месяц актуальны, с сохранением даты и пользователя подтверждения. При любых изменениях в ресурсах за соответствующий период статус автоматически сбрасывается на «не подтверждено», чтобы исключить использование устаревших данных. Это позволит руководителям подразделений видеть реальную актуальность планов в разрезе месяца.
    - **Решение:** Требует системы пользователей и ролей (см. итог). _Нужно пояснение Заказчика_ _Заявка на веб-дизайнера🙂_
- [ ] **10.** Рассмотреть целесообразность добавления валидации указанных руководителем проекта ресурсов руководителями групп ресурсов. Например, виза «Подтверждаю», которую может устанавливать/снимать руководитель, которая будет сбрасываться при актуализации плана руководителем проекта. У значения запрошенного ресурса и у визы должны фиксироваться даты последнего изменения.
    - **Решение:** Требует системы пользователей и ролей. _Нужно пояснение Заказчика_ _Заявка на веб-дизайнера🙂_
- [ ] **11.** Сейчас ресурсный план по некоторым целям не даёт понять, на что именно планируется потратить ресурсы. Решение — рассмотреть возможность создания в Confluence на странице каждой цели года укрупнённого плана проекта со списком задач-вех (разделённых по группам ресурсов) и оценивать планируемые ресурсы на эти задачи. Тогда в ресурсный план можно будет просто подгружать автоматически сумму по указанным целям года, а не проставлять вручную.
    - **Решение:** _Нужно пояснение Заказчика_
- [ ] **12.** Фильтрация проектов на странице «Управление данными» по активным проектам, по руководителю проекта
    - **Решение:** **Где:** «Управление данными» → панель фильтров (появляется при выборе группы). Добавятся фильтры: показывать только активные (не архивные) проекты, а фильр по РП уже есть. **Оценка: 5 ч**
- [ ] **13.** На вкладке Ресурсный план добавить маленькую кнопочку Актуализировано. Руководитель проекта должен ее нажать, когда актуализированы ресурсы по проекту. И при наведении показывает дату последнего нажатия (соответственно нажимать эту кнопку должны быть права только у руководителей проектов) и каждое 20-е число месяца актуализация слетает. Можно также сделать, чтобы у нее менялся цвет, чтобы было более явно видно, кто актуализировал, а кто нет
    - **Решение:** **Где:** «Ресурсный план» → в строке каждого проекта (узкая кнопка в колонке «Проект» или в отдельной колонке). Появится небольшая кнопка «Актуализировано»: руководитель проекта нажимает её, подтверждая, что обновил ресурсы; при наведении видна дата последнего нажатия, а цвет показывает — актуально или нет. Каждое 20-е число месяца отметка сбрасывается. Нажимать может только руководитель проекта — требует системы пользователей и ролей. **Оценка: 10 ч**
- [ ] **14.** Также надо в регламенте прописать, что если данные руководитель проекта актуализировал, а руководитель(_Руководитель группы?_) не согласен, то что делать?
    - **Решение:** **Где:** Не на странице сайта — документ-регламент; по желанию подсказка на «Ресурсном плане» рядом с кнопкой «Актуализировано». Нужно описать, что делать, если руководитель проекта актуализировал данные, а руководитель группы/подразделения с ними не согласен. **Оценка: 4 ч**
