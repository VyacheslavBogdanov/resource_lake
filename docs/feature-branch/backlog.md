# Доработки ресурсного плана

- [ ] **1.** Ранее мы в Excel фиксировали ошибки планирования. На сайте нет возможности указать, сколько реально потрачено, для анализа.
    - **Решение:** на странице «Управление данными» рядом с плановыми часами появится поле для ввода фактически потраченных часов. На «Ресурсном плане» добавится колонка «Факт» и колонка отклонения (план минус факт) — будет видно, где ресурсы недо- или переоценили. **Оценка: 16 ч**
- [ ] **2.** Дата обновления данных по ресурсу
    - **Решение:** возле каждого значения ресурса будет видно дату его последнего изменения (при наведении на ячейку или в отдельной колонке) — сразу понятно, насколько свежие данные. **Оценка: 6 ч**
- [ ] **3.** Добавить систему стимуляции руководителя проекта поддерживать план в актуальном состоянии
    - **Решение:** проекты с устаревшими данными будут подсвечиваться цветом, а сверху появится сводка «кто обновил план, а кто нет». Это подталкивает руководителей держать план актуальным. Пересекается с пунктами 9 и 13. **Оценка: 8 ч**
- [ ] **4.** Фильтрация строк с нулевыми значениями для выбранной группы/ресурса. Проектов много, но значимых строк бывает менее десятка — не в кайф скроллить пустые строки
    - **Решение:** на «Ресурсном плане» появится переключатель «Скрыть пустые строки» — проекты, где у выбранной группы/ресурса нет часов, временно прячутся, чтобы не листать пустые строки. **Оценка: 4 ч**
- [ ] **5.** На странице «Ресурсный план», столбец «доступно» не информативен. Лучше указывать часы, которые на данный момент доступны
    - **Решение:** в шапке колонки вместо общей мощности группы будет показываться, сколько часов ещё свободно (мощность минус уже распределённые). Перегруз (распределили больше, чем есть) подсветится. **Оценка: 4 ч**
- [ ] **6.** На вкладке «Ресурсный план» в столбце «Проект» добавить возможность выбора определенных проектов (выбор осуществляет пользователь) для отображения только их
    - **Решение:** на «Ресурсном плане» появится возможность галочками выбрать конкретные проекты и видеть в таблице только их. **Оценка: 6 ч**
- [ ] **7.** На вкладке «Ресурсный план», при наведении на проект выскакивает хинт, в нем помимо заказчика еще указывать и руководителя проекта.
    - **Решение:** во всплывающей подсказке при наведении на проект, помимо заказчика, будет показываться и руководитель проекта. **Оценка: 1 ч**
- [ ] **8.** Автоматическое сохранение введенных данных на странице «Управление данными», без нажатия кнопки «Сохранить»
    - **Решение:** на странице «Управление данными» введённые данные будут сохраняться сами, без нажатия кнопки «Сохранить». Рядом появится индикатор «Сохранение… / Сохранено», кнопку оставим на всякий случай. **Оценка: 8 ч**
- [ ] **9.** Предлагается ввести помесячную фиксацию актуальности ресурсного плана внутри каждого квартала: руководитель проекта вручную подтверждает, что данные за конкретный месяц актуальны, с сохранением даты и пользователя подтверждения. При любых изменениях в ресурсах за соответствующий период статус автоматически сбрасывается на «не подтверждено», чтобы исключить использование устаревших данных. Это позволит руководителям подразделений видеть реальную актуальность планов в разрезе месяца.
    - **Решение:** внутри квартала по месяцам появится возможность руководителю проекта отметить, что данные за месяц актуальны (с сохранением, кто и когда подтвердил). Если кто-то меняет ресурсы за этот период, отметка сама слетает в «не подтверждено». Руководители подразделений увидят общую картину актуальности по месяцам. Требует системы пользователей и ролей (см. итог). **Оценка: 32 ч**
- [ ] **10.** Рассмотреть целесообразность добавления валидации указанных руководителем проекта ресурсов руководителями групп ресурсов. Например, виза «Подтверждаю», которую может устанавливать/снимать руководитель, которая будет сбрасываться при актуализации плана руководителем проекта. У значения запрошенного ресурса и у визы должны фиксироваться даты последнего изменения.
    - **Решение:** у руководителя группы ресурсов появится возможность ставить и снимать визу «Подтверждаю» на запрошенных у его группы ресурсах. При изменении значения ресурса виза автоматически снимается. У значения и у визы будет видна дата последнего изменения. Требует системы пользователей и ролей. **Оценка: 24 ч**
- [ ] **11.** Сейчас ресурсный план по некоторым целям не даёт понять, на что именно планируется потратить ресурсы. Решение — рассмотреть возможность создания в Confluence на странице каждой цели года укрупнённого плана проекта со списком задач-вех (разделённых по группам ресурсов) и оценивать планируемые ресурсы на эти задачи. Тогда в ресурсный план можно будет просто подгружать автоматически сумму по указанным целям года, а не проставлять вручную.
    - **Решение:** на странице каждой цели года в Confluence ведётся укрупнённый план со списком задач-вех и оценкой ресурсов, а в «Ресурсный план» эти суммы подтягиваются автоматически, без ручного ввода. Крупная связка с внешней системой — нужен серверный обмен и доступ к Confluence; оценка минимальная. **Оценка: ≥40 ч**
- [ ] **12.** Фильтрация проектов на странице «Управление данными» по активным проектам, по руководителю проекта
    - **Решение:** на странице «Управление данными» появятся фильтры: показывать только активные (не архивные) проекты и фильтр по руководителю проекта. **Оценка: 5 ч**
- [ ] **13.** На вкладке Ресурсный план добавить маленькую кнопочку Актуализировано. Руководитель проекта должен ее нажать, когда актуализированы ресурсы по проекту. И при наведении показывает дату последнего нажатия (соответственно нажимать эту кнопку должны быть права только у руководителей проектов) и каждое 20-е число месяца актуализация слетает. Можно также сделать, чтобы у нее менялся цвет, чтобы было более явно видно, кто актуализировал, а кто нет
    - **Решение:** в каждой строке проекта на «Ресурсном плане» появится небольшая кнопка «Актуализировано». Руководитель проекта нажимает её, подтверждая, что обновил ресурсы; при наведении видна дата последнего нажатия, а цвет показывает — актуально или нет. Каждое 20-е число месяца отметка сбрасывается. Нажимать может только руководитель проекта — требует системы пользователей и ролей. **Оценка: 10 ч**
- [ ] **14.** Также надо в регламенте прописать, что если данные руководитель проекта актуализировал, а руководитель не согласен, то что делать?
    - **Решение:** это не доработка сайта, а пункт регламента — нужно описать, что делать, если руководитель проекта актуализировал данные, а руководитель группы/подразделения с ними не согласен. По желанию — добавить короткую подсказку прямо в интерфейсе. **Оценка: 0 ч**
